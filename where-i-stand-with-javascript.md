The extent of my knowledge of JavaScript before learning the language is that it’s very controversial among developers. A quick Google search of JavaScript will reveal many who praise it but also many who simply hate it. Thus, with that as my only understanding of the language, I was eager to learn and weigh the pros and cons myself.  

I have prior experiences with Java, C, and C++ and as a complete novice of JavaScript, having only covered the basics, I can understand why it is a preferred language to some. I find that its basic syntax is all quite familiar. JavaScript’s for-loops, while-loops, do/while loops, if/else, and switch statements are essentially the same in C, and C++. However, one thing about JavaScript that stands out is that code can be shortened, using its unique syntax. This is an advantage because it is convenient and saves time by combing many steps and condensing them into single steps so lines of code can be kept at a minimum. For example, traditional if/else statements comparing variables and constants take multiple lines of code and can be redundant to type out every time you need to utilize it. JavaScript code on the other hand, uses the ternary operator syntax to shorten multiple lines of code into one line, producing the same results. Some other new things I’ve learned about JavaScript are arrow functions, the rest parameter, default parameters, destructuring assignment, promises, and the differences between the ‘var’ and ‘let’ keywords.

Although writing shorter lines of code saves time and reduces workload, I see some disadvantages to it. My first concern is the code’s readability. Condensed code using JavaScript syntax can be difficult to follow, find errors, and maintain. Declaring multiple variables and writing functions in JavaScript can also be written in a single line of code but the syntax is less intuitive. Having multiple lines of code is easier on the eyes and very straightforward. Personally, I prefer readability over efficiency but JavaScript is still new to me and with time and practice, I might have a different perspective. 

I used freeCodeCamp to learn the basics of JavaScript and its ES6 features, and it’s an amazing website that really guided me in grasping concepts by actually writing code. I also find that practicing WODs are extremely helpful because they force me to think and solve under pressure. Repeating WODs allow me to practice problem solving and aids in retaining information. Although a bit stressful, I believe this style of learning is beneficial for me in becoming a more successful programmer. With practice, I believe Javascript could be a preferred language.
